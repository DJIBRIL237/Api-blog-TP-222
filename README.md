API Blog - INF222 Backend Ce projet consiste en une API Backend pour la gestion d'un blog simple, développée avec Node.js (Express) et MongoDB Atlas. Il a été structuré en suivant le parcours d'apprentissage CleeRoute.🌿 Structure des BranchesLe dépôt est organisé en deux branches pour séparer les environnements :master : Version de développement local. Utilise une configuration adaptée à votre machine (ex: base de données locale ou variables d'environnement de test).deploy : Version de production configurée pour le déploiement sur Railway. Cette branche contient les variables d'environnement (MONGODB_URI) nécessaires à la connexion avec MongoDB Atlas en ligne.🚀 Installation et LancementPrérequisNode.js installé.Un compte MongoDB Atlas (ou une instance locale).Installation locale (Branche master)Clonez le dépôt :Bashgit clone [LIEN_DE_VOTRE_DEPOT]
git checkout master
Installez les dépendances :Bashnpm install
Créez un fichier .env à la racine et ajoutez votre lien de connexion :Extrait de codeMONGODB_URI=votre_lien_mongodb
PORT=3000
Lancez le serveur :Bashnpm start
🛠 Architecture du ProjetL'application suit une séparation claire des responsabilités pour garantir une maintenance facile:Models : Définition des schémas de données MongoDB (Titre, Contenu, Auteur, etc.).Controllers : Logique métier et traitement des requêtes.Routes : Définition des points d'entrée de l'API.📑 Documentation de l'API (Endpoints)L'API est documentée via Swagger. Une fois le serveur lancé, accédez à la documentation sur : https://[votre-url-railway].app/api-docs/.Liste des routes:MéthodeEndpointDescriptionPOST/api/articlesCréer un nouvel article GET/api/articlesRécupérer tous les articles GET/api/articles/{id}Récupérer un article par son ID PUT/api/articles/{id}Modifier un article existant DELETE/api/articles/{id}Supprimer un article GET/api/articles/searchRechercher par titre ou contenu (?query=...) Codes de statut HTTP utilisés:200 OK : Succès de la requête.201 Created : Création réussie d'un article.400 Bad Request : Données manquantes ou mal formées.404 Not Found : Article inexistant.500 Internal Server Error : Erreur côté serveur.🌐 DéploiementLa version en ligne est hébergée sur Railway.Lien API/Swagger : [Insérez votre lien Railway ici]
